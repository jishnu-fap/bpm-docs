Admin
Edit a user
Click the kebab menu (⋮) on a row and select Edit. Change the role, department, group, or name. You can also trigger a password reset from the edit page — the user receives an email with a reset link.
Changing a user's role takes effect immediately — their permissions update on their next page load without needing to log out.

Clone permissions from another user
From the kebab menu select Clone Permissions. Pick a source user — the new user receives the exact same role assignment. Useful when onboarding someone into an existing team position.

6
Admin
Deactivate a user
From the kebab menu select Deactivate. The user can no longer log in. Their history, assignments, and audit records are preserved. They can be reactivated at any time from the Users list by switching the status filter to Inactive.
Deactivating does not unassign the user from projects or checklists — review those separately before offboarding.
